Messages Example

This will stream random noise data from ROS, and optionally start and stop the random noise broadcasting on the ROS side.

To run on the ROS side, run the following commands:

  • roscore
  • roslaunch rosbridge_server rosbridge_websocket.launch
  • rosrun basics noise.py

The first command spins up ROS, the second command will publish the ROS bridge server which allows for communication through websockets, and the third command broadcasts random noise data on the /chatter topic.

Below, the random noise is written into the output div. The button will broadcast the messages "start" and "stop", which are interpreted by the /chatter topic to start and stop broadcasting random data.

Sometimes, commands can take a while to be caught by ROS, so give it a sec or two.
